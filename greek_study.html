BACKGROUND

Greek Study is a mobile application used by college Greek organizations to keep track of how many study hours each member has completed weekly. Users must be at designated areas on campus in order for their hours to be taken accounted for.

OBJECTIVE

Redesign the interface of Greek Study to improve the overall user experience of the current app.

USER RESEARCH

I wanted to get more insight on other people's experiences using the app. In addition to reading over Greek Study's ratings and reviews in the app store, I reached out to both frequent and first-time users to see what issues they have encountered upon using the application.
These were the key pain points that I found:
1. Users have a hard time figuring out how many hours they have completed for each week.
2. Users find the main menu to be confusing and poorly designed.
3. Users want all functions to be stored under one main menu.
4. Users prefer the app to automatically start counting hours when the user is in a study location rather than having to press on a button every time because they tend to forget to do so, thus losing hours.

PERSONA

After conducting user research, I summarized my key findings into a target user persona.

IDEATION

From the feedback gained through user research, I found that many of the issues fall into two main categories.
This is what I came up with as possible solutions for these problems:
Organization
- Revise the login page to eliminate clutter.
- Modify the menu and placing it at the top of the screen.
- Display the list of locations in alphabetical order to make them easier to find.
Automation
- Get rid of the "Start Studying" button and enforce an automatic timer that starts when the user is in a designated location.
- Add a total hour count feature for each week.
